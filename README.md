# FlyB — موقع مقارنة أسعار الطيران والفنادق

موقع مقارنة أسعار (طيران + فنادق) بالعربية والإنجليزية. الموقع نفسه لا يبيع تذاكر
ولا يستقبل مدفوعات — دوره المقارنة ثم تحويل العميل لوكيل سفر حقيقي لإكمال الحجز.

| الموقع (GitHub Pages) | https://itqissam26.github.io/flyB/ |
| السيرفر الخلفي (Railway) | https://flyb-production-e253.up.railway.app |

## البنية

- **`index.html`** — الموقع كامل بملف واحد (HTML/CSS/JS، بدون أي إطار عمل أو خطوة بناء).
  الصور مضمّنة Base64 داخل الملف.
- **`server/`** — سيرفر Express صغير، وظيفته إخفاء توكن Travelpayouts وجلب أرخص سعر حقيقي.
  منشور على Railway، ويعيد النشر تلقائيًا عند كل push للفرع `main`.

## الربط مع Travelpayouts (شبكة العمولة)

| العنصر | الحالة |
|---|---|
| رقم الشريك (marker) | `751090` — مضمّن في `index.html` |
| سكربت Drive | مركّب في `<head>` — يحوّل روابط السفر لمربحة تلقائيًا |
| ودجت بحث Aviasales | مضمّن داخل قسم "ابحث واحجز مباشرة من شريكنا" |
| توكن الـ API | في `server/.env` محليًا + متغير بيئة على Railway (**غير مرفوع لـ git**) |

## وضع الأسعار الحالي — مهم

- **الطيران:** الأسعار المعروضة تجريبية (مولّدة برمجيًا)، لكنها **تُضبط على أرخص سعر حقيقي**
  يُجلب من Travelpayouts Data API، فتظهر شارة "أسعار حية".
- **الفنادق:** تجريبية بالكامل. واجهة Hotellook القديمة (`engine.hotellook.com`) تم
  إيقافها من طرفهم (404 على كل المسارات)، والمسار معطّل صراحة في السيرفر.
- **القيد الجوهري:** كروت المقارنة بيانات مولّدة (شركة الطيران/الوقت/رقم الرحلة غير حقيقية)،
  لذلك **لا يمكن** ربط كرت معيّن برحلة حقيقية محددة عند الوسيط.

## الخطوة التالية (متوقفة مؤقتًا — نستأنف لاحقًا)

الهدف: تجربة مثل **سكاي سكانر / وي قو** — بحث → رحلات حقيقية → مقارنة وكلاء لكل
رحلة → اختيار وكيل → إكمال الحجز عنده.

الطريق الوحيد لتحقيقها: **Travelpayouts White Label** (محرك بحث وحجز كامل جاهز
تحت اسمك). تم اتخاذ القرار باعتماده.

**المراحل الأربع:**

1. **شراء نطاق** ⬅️ *وقفنا هنا* — (~٦٠ ريال/سنة، من Cloudflare أو Namecheap).
   مرشّحات: `flyb.net` / `flyb.co` / `getflyb.com`.
   White Label **لا يعمل** على نطاق `github.io`، لذا النطاق شرط أساسي.
2. **إنشاء White Label** من لوحة Travelpayouts → White Label Web → Create White Label،
   وربطه بنطاق فرعي مثل `search.<النطاق>`.
3. **ضبط DNS:** سجل CNAME للنطاق الفرعي ← `whitelabel.travelpayouts.com`،
   وسجل CNAME للنطاق الرئيسي ← `itqissam26.github.io`. (قد يستغرق حتى ٧٢ ساعة)
4. **الربط النهائي بالكود:** توجيه زر البحث في الموقع للمحرك، وربط النطاق بـ GitHub Pages.

**الشكل النهائي المستهدف:** النطاق الرئيسي = الموقع الحالي بتصميمه،
والنطاق الفرعي = محرك الحجز الحقيقي، وزر يربط بينهما.

## تشغيل السيرفر محليًا

```bash
cd server
npm install
# أنشئ ملف .env من .env.example وضع فيه التوكن
npm run dev     # http://localhost:8787
```

> ملاحظة: `index.html` يشير حاليًا لسيرفر Railway المنشور، لا للمحلي.
> لتجربة السيرفر المحلي، غيّر `API_BASE` في `index.html` مؤقتًا.
